 Context

 Currently all client reads are routed to the primary node for each slot via check_moved() in redis.rs. Replica nodes receive replicated data but never
 serve client reads. This wastes replica capacity and increases read latency. Redis Cluster solves this with READONLY/READWRITE commands — a per-connection
  flag that lets clients opt into reading from replicas (with eventual consistency).

 Changes

 Step 1: Per-connection readonly flag in RedisHandler

 File: src/server/redis.rs

 - Add readonly: bool field to RedisHandler (line 359)
 - Add READONLY command → sets self.readonly = true, responds +OK
 - Add READWRITE command → sets self.readonly = false, responds +OK
 - Add both to the command dispatch table (line 402)

 Since handle_command takes &mut self (it doesn't — it takes &self), we need to change readonly to a Cell<bool> or make handle_command take &mut self.
 Looking at the code, handle_command takes &self. The simplest approach: use std::cell::Cell<bool> for interior mutability (single-threaded per-connection,
  no need for atomic).

 Step 2: Modify check_moved for readonly replica reads

 File: src/server/redis.rs (line 441)

 Create check_moved_read(&self, key, out) -> bool for read commands:
 - If self.readonly is true AND this node is a replica for the key's slot → return false (allow local read)
 - If self.readonly is true but this node is neither primary nor replica → still send MOVED
 - If self.readonly is false → delegate to existing check_moved() (primary-only)

 This requires access to topology from RedisHandler. The router already has topology. Add a helper:
 fn is_local_replica_for_key(&self, key: &[u8]) -> bool {
     if let Some(router) = &self.router {
         let slot = ClusterTopology::slot_for_key(key);
         let topo = router.topology().read();
         topo.is_local_replica(slot)
     } else {
         false
     }
 }

 Update read commands (cmd_get, cmd_mget, cmd_exists, cmd_ttl, cmd_pttl, cmd_dbsize) to use check_moved_read instead of check_moved.

 Write commands (cmd_set, cmd_del, cmd_mset, cmd_expire, cmd_pexpire, cmd_persist, cmd_expireat, cmd_pexpireat) continue using check_moved (writes always
 go to primary).

 Step 3: Update CLUSTER NODES to show replica flags

 File: src/server/redis.rs (line 919, cmd_cluster NODES subcommand)

 Currently all nodes are shown as "master". Change to:
 - For each node, check if it's a primary for any slot → "master"
 - If it only appears in replicas lists → "slave" with <primary_id> as the master link
 - Redis format: <id> <ip>:<port>@<cport> slave <master_id> 0 0 <version> connected

 For simplicity (since all nodes are both primary for some slots and replica for others in our setup), output each node as "master" for the slots it owns
 as primary. This matches Redis behavior where a node can be master for some slots.

 Step 4: Add --replica-read CLI flag

 File: src/config.rs

 Add optional config flag:
 /// Allow replica nodes to serve read requests (clients must still send READONLY)
 #[arg(long)]
 pub replica_read: bool,
 Default: false. When false, READONLY command is accepted but check_moved_read still sends MOVED (server-side disable). This gives operators control.
