│                                                                                                                                                          │
│ Context                                                                                                                                                  │
│                                                                                                                                                          │
│ The SET command currently only supports EX (seconds) and PX (milliseconds) TTL options. Redis clients expect full SET option support including EXAT,     │
│ PXAT, NX, XX, KEEPTTL, and GET. Additionally, the server has zero capacity management — writes succeed indefinitely until the OS runs out of             │
│ disk/memory, with no eviction or expired-entry cleanup.                                                                                                  │

│ Replace the current parse_set_options(&self, args) -> u32 with a richer struct:                                                                          │
│                                                                                                                                                          │
│ struct SetOptions {                                                                                                                                      │
│     ttl_secs: u32,    // 0 = no TTL (computed from EX/PX/EXAT/PXAT)                                                                                      │
│     nx: bool,         // Only set if NOT exists                                                                                                          │
│     xx: bool,         // Only set if EXISTS                                                                                                              │
│     keepttl: bool,    // Retain existing TTL                                                                                                             │
│     get: bool,        // Return old value                                                                                                                │
│ }                                                                                                                                                        │
│                                                                                                                                                          │
│ Parse all options from args slice in a single pass:                                                                                                      │
│ - EX secs → ttl_secs = secs (already works)                                                                                                              │
│ - PX ms → ttl_secs = ms / 1000 (already works)                                                                                                           │
│ - EXAT timestamp → ttl_secs = timestamp - now_secs (new)                                                                                                 │
│ - PXAT timestamp_ms → ttl_secs = (timestamp_ms - now_ms) / 1000 (new)                                                                                    │
│ - NX → nx = true                                                                                                                                         │
│ - XX → xx = true                                                                                                                                         │
│ - KEEPTTL → keepttl = true                                                                                                                               │
│ - GET → get = true                                                                                                                                       │
│                                                                                                                                                          │
│ Update cmd_set logic                                                                                                                                     │
│                                                                                                                                                          │
│ After parsing key, value, and options:                                                                                                                   │
│                                                                                                                                                          │
│ 1. If opts.nx: check handler.get_value(key) — if exists, return $-1\r\n (nil)                                                                            │
│ 2. If opts.xx: check handler.get_value(key) — if NOT exists, return $-1\r\n (nil)                                                                        │
│ 3. If opts.get: read old value via handler.get_value(key) before writing                                                                                 │

│ 5. Call handler.put_sync(key, value, final_ttl)                                                                                                          │
│ 6. If opts.get: return old value (or nil). Otherwise return +OK                                                                                          │

│ Step 3: Eviction Engine                                                                                                                                  │
│                                                                                                                                                          │
│ New file: src/storage/eviction.rs                                                                                                                        │
│                                                                                                                                                          │
│ EvictionPolicy enum                                                                                                                                      │
│                                                                                                                                                          │
│ pub enum EvictionPolicy {                                                                                                                                │
│     NoEviction,       // Return error when full                                                                                                          │
│     AllKeysLru,       // Evict least-recently-used across all keys                                                                                       │
│     VolatileLru,      // Evict LRU among keys with TTL                                                                                                   │
│     AllKeysRandom,    // Evict random keys                                                                                                               │
│     VolatileRandom,   // Evict random keys with TTL                                                                                                      │
│     VolatileTtl,      // Evict keys with shortest remaining TTL                                                                                          │
│ }                                                                                                                                                        │
│                                                                                                                                                          │
│ EvictionConfig                                                                                                                                           │
│                                                                                                                                                          │
│ pub struct EvictionConfig {                                                                                                                              │
│     pub policy: EvictionPolicy,                                                                                                                          │
│     pub max_entries: u64,       // 0 = unlimited                                                                                                         │
│     pub max_data_mb: u64,       // 0 = unlimited                                                                                                         │
│     pub check_interval_secs: u64,                                                                                                                        │
│     pub sample_size: usize,     // Keys to sample per eviction pass (default: 16)                                                                        │
│ }                                                                                                                                                        │
│                                                                                                                                                          │
│ Evictor struct                                                                                                                                           │
│                                                                                                                                                          │
│ Follows the same pattern as Compactor:                                                                                                                   │
│                                                                                                                                                          │
│ pub struct Evictor {                                                                                                                                     │
│     config: EvictionConfig,                                                                                                                              │
│     handler: Arc<Handler>,                                                                                                                               │
│     index: Arc<Index>,                                                                                                                                   │
│     file_manager: Arc<FileManager>,                                                                                                                      │
│     stats: Arc<EvictionStats>,                                                                                                                           │
│ }                                                                                                                                                        │
│                                                                                                                                                          │
│ Core methods:                                                                                                                                            │
│                                                                                                                                                          │
│ - run(stop: Arc<AtomicBool>) — background loop, same pattern as compaction thread                                                                        │
│ - should_evict() -> bool — checks if capacity thresholds are exceeded                                                                                    │
│ - evict_batch() -> usize — evicts entries until below threshold, returns count evicted                                                                   │
│ - evict_expired() -> usize — removes expired entries (runs every pass regardless of capacity)                                                            │
│ - select_victim(shard: &IndexShard) -> Option<key> — policy-specific victim selection                                                                    │
│                                                                                                                                                          │
│ Expired entry cleanup                                                                                                                                    │
│                                                                                                                                                          │
│ Every cycle, scan a portion of shards (round-robin, e.g. 16 shards per cycle) and:                                                                       │
│ 1. For each live entry, check if the on-disk record has expired using handler.get_with_meta(key)                                                         │
│ 2. If expired: call index.remove(key) to free memory                                                                                                     │
│                                                                                                                                                          │
│ This is lightweight because get_with_meta checks write buffer first (hot path) and only goes to disk for cold entries.                                   │
│                                                                                                                                                          │
│ LRU approximation (for allkeys-lru / volatile-lru)                                                                                                       │
│                                                                                                                                                          │
│ Since IndexEntry has no access-time field, use random sampling (Redis-style approximation):                                                              │
│ 1. Pick sample_size random entries from random shards                                                                                                    │
│ 2. For LRU: pick the one with the lowest generation (generation is monotonically increasing, so lower = older write, approximation of LRU)               │
│ 3. For volatile variants: filter to entries whose on-disk record has TTL > 0                                                                             │
│ 4. Delete the victim via handler.delete_sync(key) + index.remove(key)                                                                                    │
│                                                                                                                                                          │
│ volatile-ttl                                                                                                                                             │
│                                                                                                                                                          │
│ Sample random entries with TTL, pick the one closest to expiry (lowest timestamp + ttl). Requires get_with_meta to read the TTL from disk/write-buffer.  │
│                                                                                                                                                          │
│ Capacity check in write path                                                                                                                             │
│                                                                                                                                                          │
